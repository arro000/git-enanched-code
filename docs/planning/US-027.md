## User Story

**Epic:** EP-002 — Layout 3 Colonne & Monaco Editor
**Priorità:** HIGH | **Story Points:** 3

**Story**
Come developer,
voglio che l'interfaccia del Merge Editor corrisponda al mockup visivo definito in `docs/mockups/merge-editor.html`,
in modo da avere un'esperienza utente coerente e professionale con il design system VS Code Dark+.

**Criteri di Accettazione**
- [ ] La toolbar mostra un conflict badge con pulsating dot + "N conflicts remaining" aggiornato in tempo reale
- [ ] Le intestazioni colonne mostrano color-bar verticale (amber/teal/blue), nome colorato, branch badge e tag "read-only"/"editable"
- [ ] Le conflict zone usano la palette VS Code Dark+: HEAD amber, MERGING blue, RESULT pending rosso, RESULT resolved teal
- [ ] I pulsanti azione conflitto corrispondono al mockup: `>> Accept Current` (amber), `<< Accept Incoming` (blue), `✕ Ignore` (neutro)
- [ ] Una minimap strip da 14px compare sul lato destro con segmenti colorati per i conflitti
- [ ] Una status bar VS Code-style in fondo mostra il nome file, contatore conflitti e shortcut F7/Shift+F7

## Analisi dei Requisiti

> **Analista:** Emanuele 🔎

### Scope

Il mockup `docs/mockups/merge-editor.html` definisce un design system VS Code Dark+ completo. L'implementazione corrente usa variabili CSS generiche di VS Code (`--vscode-foreground`, ecc.) che producono un look generico e non corrispondente al mockup. La storia copre la sostituzione visiva mantenendo intatta tutta la logica funzionale.

**Incluso:**
- Riscrittura CSS con palette VS Code Dark+ hardcoded (HEAD amber, RESULT teal, MERGING blue)
- Toolbar con merge badge animato e pulsanti primary/secondary stilizzati
- Column headers con color-bar, branch info, colored names, tag editable/read-only
- Conflict zone con styled backgrounds e border-left colorati
- Conflict action buttons con label estese e stili per stato
- Minimap strip statica 14px con segmenti colorati
- Status bar con branch info e keyboard shortcuts

**Escluso:**
- Tab bar multi-file (non funzionale nell'estensione)
- Minimap interattiva con click-to-navigate (EP-004, US-020)
- Auto-resolve button funzionale (EP-003, US-014)
- Breadcrumb navigabile

### Requisiti Impliciti

- Il conflict counter nel toolbar deve aggiornarsi ogni volta che un conflitto viene gestito
- Il branch name nelle column headers è estratto dal fileName (placeholder se non disponibile)
- Il minimap deve mostrare segmenti rossi per conflitti aperti, teal per risolti, come da mockup

### Assunzioni

- La palette VS Code Dark+ è hardcoded (non si usa `var(--vscode-*)`) per produrre esattamente il look del mockup; questo è accettabile perché il merge editor è sempre usato in un contesto di codice dove Dark+ è standard.
- Il branch name nei column headers mostra il nome del file come breadcrumb, non richiede git API.
- La minimap è statica (segmenti proporzionali ai conflitti, non interattiva).

## Soluzione Tecnica

> **Architetto:** Leonardo 📐

### Approccio Scelto

Riscrittura della sezione `<style>` e dell'HTML strutturale in `getMergeEditorHtml()` dentro `src/ui/MergeEditorProvider.ts`. La logica JavaScript (Monaco Editor, gestione chunk, messaggi VS Code API) è mantenuta intatta con aggiunte minimali per il conflict counter nel toolbar.

### Motivazione

Cambio localizzato in un singolo metodo, zero rischio di regressione sulla logica funzionale. La palette hardcoded è la stessa del mockup approvato.

### Modifiche alle API
