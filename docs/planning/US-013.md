## User Story

**Epic:** EP-003 — Smart Merge Engine
**Priorità:** HIGH | **Story Points:** 5

**Story**
Come Marco (Senior Developer),
voglio che Tree-sitter analizzi i conflitti rimanenti dopo diff3 e proponga risoluzioni semanticamente corrette,
in modo da risolvere automaticamente anche i conflitti che diff3 non riesce a gestire senza necessità di review manuale caso per caso.

**Criteri di Accettazione**
- [ ] Tree-sitter analizza la struttura AST del codice nei conflitti non risolti da diff3
- [ ] Se le modifiche sono semanticamente compatibili (es. aggiunta di metodi in classi diverse, import non sovrapposti), viene proposta una risoluzione con score di confidenza calcolato
- [ ] L'analisi AST si completa entro 1s per file fino a 5.000 righe

---

## Analisi dei Requisiti

> **Analista:** Emanuele 🔎

### Scope

- L'analisi AST è il Layer 2 dello Smart Merge Engine, opera sui conflitti NON risolti dal Layer 1 (diff3)
- Richiede parsing del codice HEAD e MERGING in alberi sintattici tramite Tree-sitter
- Identifica pattern semanticamente compatibili: import indipendenti, metodi/funzioni aggiunti in posizioni diverse, proprietà non sovrapposte
- Calcola uno score di confidenza per ogni risoluzione proposta
- Per questa story, il supporto linguistico è limitato a TypeScript/JavaScript (US-016 estenderà agli altri linguaggi)

### Requisiti Impliciti

- Nuovo valore `'ast-auto'` per il tipo union `sorgenteApplicata` in `StatoRisoluzioneConflitto`
- Se il linguaggio del file non è supportato, l'analisi AST viene saltata silenziosamente
- I conflitti senza BASE rimangono irrisolti (non candidabili per AST)
- La libreria `web-tree-sitter` (WASM-based) deve essere aggiunta come dipendenza
- Le grammar Tree-sitter per TypeScript e JavaScript devono essere incluse come file `.wasm`

### Assunzioni

- L'uso di `web-tree-sitter` è la scelta corretta per VS Code extensions (WASM, no native bindings)
- I file `.wasm` delle grammar vengono bundlati con l'estensione
- Lo score di confidenza è un numero 0.0–1.0: >= 0.8 alta, 0.5–0.8 media, < 0.5 bassa

## Soluzione Tecnica

> **Architetto:** Leonardo 📐

### Approccio Scelto

Creare un servizio `AnalizzatoreAstConflitti` che usa `web-tree-sitter` per parsare le sezioni HEAD e MERGING dei conflitti irrisolti, identifica pattern semantici compatibili tramite `RilevatorePatternSemantico`, e propone risoluzioni con confidenza.

### Motivazione

- `web-tree-sitter` è la libreria standard per analisi AST in ambienti JavaScript/WASM, compatibile con VS Code
- Separare il rilevamento dei pattern dal servizio principale permette di estendere facilmente i linguaggi supportati (US-016)
- Il calcolo della confidenza integrato nei pattern permette risoluzioni granulari

### Pattern Semantici Supportati

1. **Import indipendenti**: HEAD e MERGING aggiungono import diversi → merge entrambi
2. **Funzioni/metodi aggiunti**: entrambi aggiungono funzioni/metodi diversi nello stesso scope → merge entrambi
3. **Proprietà oggetto/classe**: entrambi aggiungono proprietà diverse → merge entrambi
4. **Blocchi di codice indipendenti**: modifiche in scope diversi dell'albero sintattico → merge

### Interfacce Principali

```typescript
interface RisultatoAnalisiAst {
    conflittiAnalizzati: RisultatoRisoluzioneAst[];
    numeroRisoltiAst: number;
    tempoEsecuzioneMs: number;
}

interface RisultatoRisoluzioneAst {
    indiceConflitto: number;
    risolvibileAutomaticamente: boolean;
    contenutoRisolto: string | null;
    scoreConfidenza: number;  // 0.0 - 1.0
    patternRilevato: string | null;  // nome del pattern semantico
}
```

### Integrazione in MergeEditorProvider

Dopo il diff3, se ci sono conflitti non risolti:
```
risultatoDiff3 = diff3Resolver.risolviConflitti(conflittiParsati)
↓
conflittiNonRisolti = filtra conflitti con risolvibileAutomaticamente === false
                      E motivoNonRisolto === 'sovrapposizione-modifiche'
↓
risultatoAst = analizzatoreAst.analizzaConflitti(conflittiNonRisolti, linguaggioId)
↓
Aggiorna stato con sorgenteApplicata: 'ast-auto'
```
