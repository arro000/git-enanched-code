## User Story

**Epic:** EP-006 — CI/CD & Distribuzione
**Priorità:** LOW | **Story Points:** 2

**Story**
Come contributor open source,
voglio trovare un CONTRIBUTING.md completo con setup locale, architettura e guida per aggiungere grammar Tree-sitter,
in modo da poter contribuire al progetto autonomamente senza dover chiedere aiuto ai maintainer.

**Criteri di Accettazione**
- [ ] CONTRIBUTING.md include le istruzioni passo-passo per il setup locale dell'ambiente di sviluppo
- [ ] CONTRIBUTING.md include una descrizione dell'architettura dei moduli principali (`core/git`, `core/merge`, `ui/webview`)
- [ ] CONTRIBUTING.md include una guida per aggiungere una nuova grammar Tree-sitter con esempio concreto

---

## Soluzione Tecnica

La storia richiede la creazione di un singolo file documentale `CONTRIBUTING.md` nella root del progetto. Il file deve essere completo, accurato rispetto alla struttura reale del codice e immediatamente utile a un contributor esterno.

- Creare `CONTRIBUTING.md` nella root con tre sezioni principali corrispondenti ai tre AC: setup locale, architettura moduli, guida Tree-sitter
- Leggere i file sorgente chiave (`LanguageDetector.ts`, `AnalizzatoreAstConflitti.ts`, `CLAUDE.md`, `package.json`) per documentare accuratamente l'architettura reale
- Includere un esempio concreto di aggiunta di una nuova grammar Tree-sitter (es. Go o Python) con i file da modificare e il codice da aggiungere
- Documentare le convenzioni del progetto (struttura test, build commands, naming) coerentemente con CLAUDE.md

---

## Strategia di Test

Poiche' la storia e' puramente documentale, la strategia di test si concentra sulla verifica strutturale del file creato.

- Un test unitario verifica che `CONTRIBUTING.md` esista nella root del progetto
- Il test verifica la presenza delle tre sezioni obbligatorie (setup locale, architettura, guida Tree-sitter)
- Il test verifica che il file non sia vuoto e contenga riferimenti ai moduli reali del progetto
