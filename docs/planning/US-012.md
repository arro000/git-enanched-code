## User Story

**Epic:** EP-003 — Smart Merge Engine
**Priorità:** HIGH | **Story Points:** 3

**Story**
Come Marco (Senior Developer),
voglio che le modifiche non sovrapposte vengano identificate e risolte automaticamente tramite diff3 prima che interagisca con l'editor,
in modo da trovare la colonna centrale già parzialmente popolata con le risoluzioni ovvie e concentrarmi solo sui conflitti genuinamente complessi.

**Criteri di Accettazione**
- [ ] All'apertura dell'editor, diff3 analizza il file e identifica le modifiche non sovrapposte tra HEAD, BASE e MERGING
- [ ] Le modifiche non sovrapposte vengono pre-applicate nella colonna centrale con confidenza massima, senza intervento dell'utente
- [ ] L'analisi diff3 si completa entro 200ms per file fino a 5.000 righe

## Analisi dei Requisiti

> **Analista:** Emanuele 🔎

### Scope

- Il diff3 è il Layer 1 dello Smart Merge Engine: risolve automaticamente le modifiche non sovrapposte
- Richiede la componente BASE (markers `|||||||`) per funzionare — senza BASE il conflitto resta aperto
- L'auto-resolve avviene **prima** dell'invio dati alla webview, all'apertura dell'editor
- I conflitti auto-risolti aggiornano lo stato di sessione con `sorgenteApplicata: 'diff3-auto'`

### Requisiti Impliciti

- Se `base === null` (conflitto 2-way), il diff3 non può risolvere → conflitto resta aperto
- Il contatore conflitti deve riflettere solo quelli non auto-risolti
- L'auto-resolve non deve interferire con il ripristino di stato esistente (sessioni salvate)
- Serve un nuovo valore `'diff3-auto'` nel tipo union di `sorgenteApplicata`

### Assunzioni

- L'algoritmo LCS con DP è sufficiente per rispettare i 200ms (conflitti tipicamente piccoli, decine di righe)
- L'interfaccia ConflictBlock può essere importata direttamente da ConflictParser.ts

## Soluzione Tecnica

> **Architetto:** Leonardo 📐

### Approccio Scelto

Servizio puro `Diff3Resolver` che implementa un merge a tre vie basato su LCS (Longest Common Subsequence). Il servizio non dipende da VS Code API, riceve `ConflictBlock[]` e restituisce risultati di risoluzione.

### Motivazione

- Servizio puro → testabilità completa senza mock
- Algoritmo LCS standard → affidabile e ben compreso
- Integrazione minima in MergeEditorProvider → basso rischio di regressione

### Componenti Impattati

| Componente | Tipo Modifica | Descrizione |
|---|---|---|
| `src/core/merge/Diff3Resolver.ts` | Nuovo | Servizio diff3 con algoritmo LCS e merge a tre vie |
| `src/core/merge/MergeSessionStateManager.ts` | Modifica | Aggiunta `'diff3-auto'` al tipo union `sorgenteApplicata` |
| `src/ui/MergeEditorProvider.ts` | Modifica | Integrazione diff3 nel flusso di apertura editor |
| `test/unit/core/merge/Diff3Resolver.test.ts` | Nuovo | Test unitari completi del servizio |

### Algoritmo

1. **Guardia base assente**: se `base === null`, ritorna `risolvibileAutomaticamente: false`
2. **Split in righe**: spezza head, base, merging per `\n`
3. **Diff lineare HEAD vs BASE**: identifica modifiche/aggiunte/rimozioni di HEAD rispetto alla base
4. **Diff lineare MERGING vs BASE**: identifica modifiche di MERGING rispetto alla base
5. **Rilevamento sovrapposizioni**: se entrambi modificano le stesse righe → non risolvibile
6. **Merge a tre vie**: se nessuna sovrapposizione, unisce le modifiche prendendo da HEAD dove HEAD ha modificato, da MERGING dove MERGING ha modificato, dalla base dove nessuno ha modificato

### Integrazione in MergeEditorProvider

```
resolveCustomTextEditor()
  ├─ parseConflicts() → ConflictBlock[]
  ├─ recuperaStato() → statoEsistente?
  │     ├─ SÌ: usa stato salvato (invariato)
  │     └─ NO: creaStatoIniziale()
  │              ├─ [NUOVO] diff3Resolver.risolviConflitti(conflitti)
  │              ├─ [NUOVO] aggiorna statiConflitti auto-risolti
  │              └─ salvaStato()
  └─ webviewPronta → inizializzaLayout + conflittiAutoRisolti
```

### Casi di Test Diff3Resolver

| Caso | Aspettativa |
|------|-------------|
| ConflictBlock con `base: null` | Non risolvibile, motivo: `base-assente` |
| HEAD modifica riga 1, MERGING modifica riga 5 | Risolvibile, contenuto unito |
| Entrambi modificano la stessa riga | Non risolvibile, motivo: `sovrapposizione-modifiche` |
| Solo HEAD modifica (MERGING = BASE) | Risolvibile, risultato = HEAD |
| Solo MERGING modifica (HEAD = BASE) | Risolvibile, risultato = MERGING |
| Aggiunte non sovrapposte in posizioni diverse | Risolvibile, entrambe presenti |
| Rimozioni non sovrapposte | Risolvibile, entrambe rimosse |
| Rimozione vs modifica stessa riga | Non risolvibile |
| HEAD/BASE/MERGING tutti vuoti | Risolvibile, contenuto vuoto |
| Array di ConflictBlock misti | Ogni blocco indipendente |
| Conflitto ~500 righe per lato | Completa sotto 200ms |
